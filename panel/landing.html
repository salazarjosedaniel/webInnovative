Cómo funciona

Flujo simple en tienda + control de operación (cuando aplica) desde la plataforma.

01

Escaneo

El cliente escanea el código de barras con lector 2D USB.

02

Consulta REST API (ERP/POS del cliente)

El visor consulta el ERP/POS del cliente vía HTTPS/REST. El endpoint puede estar en red local o cloud.

03

Respuesta del ERP/POS

Respuesta JSON optimizada: precio, descripción, impuestos/promos (y stock opcional).

04

Visualización

UI rápida en pantalla LCD 7" (producto, precio, promos, avisos).

05

Registro y diagnóstico

Logs y estado del dispositivo para soporte (básico o avanzado según plan).

06

Plataforma (Premium)

Panel de control, licencias y OTA remota para administrar flotas sin visitas.

Integración REST API (Local o Cloud)

iVisor se conecta al sistema que ya tiene el cliente. No obliga a migrar datos ni a cambiar de ERP/POS.

🏢

ERP/POS Local (On‑Premise)

Los precios viven en el servidor local del cliente. El visor consulta un endpoint dentro de la red (o vía gateway/VPN según el caso).

☁

ERP/POS en la nube

Si el cliente ya usa un ERP/POS cloud, el visor consulta la API pública segura (HTTPS) o un middleware autorizado.

🧩

Middleware del integrador (opcional)

Si el cliente no expone API, el integrador puede crear un servicio REST estándar que traduzca datos del ERP y entregue el JSON a iVisor.

Preguntas frecuentes

Resolvemos lo típico antes de iniciar.

¿Dónde están los precios: en la nube o en el sistema del cliente?

En el sistema del cliente (ERP/POS), ya sea local (on‑premise) o cloud. iVisor consulta por REST API (HTTPS). La nube de iVisor es para panel, licencias, monitoreo y OTA.

¿Necesito cambiar de ERP/POS para usar iVisor?

No. iVisor se integra al sistema existente mediante un endpoint REST (del cliente o del integrador).

¿La integración es solo cloud?

No. Puede ser con ERP local (red interna) o cloud. En casos on‑premise, se define el mecanismo de conectividad (gateway/VPN/proxy) según la política del cliente.

¿Qué incluye la suscripción Premium?

Panel de control, licenciamiento, logs, monitoreo y actualizaciones OTA remotas. No es para “guardar precios”.
